ECMAScript介绍

ECMAScript 是一种开放的、国际上广为接受的脚本语言规范。 它本身并不是一种脚本语言。
ECMAScript与Web浏览器没有依赖关系，ECMA定义的只是这门语言的基础，在此基础上可以构建更完善的脚本语言（JavaScript），Web浏览器只是ECMA的宿主环境之一。

  • Undefined类型

  Undefined类型只有一个值，即特殊的undifined,在使用var声明变量时，如果没有对其初始化，其变量的值就是undefined

  • Null类型

  Null类型是一个只有一个值的数据类型，即特性的值null，它表示一个空对象引用(指针)，而typeof操作符检测null会返回object。

  如果定义的变量准备在将来用于保存对象，那么最好将该变量初始化为null,这样检查null值就知道是否变量已经分配了对象引用

  • Boolean类型

  Boolean类型有两个字面值:true和false，虽然字面量只有两种，但所有类型的值都有与这连个等价的值，可以进行转换。

  1. 显式转换:使用转型函数Boolean():

  2. if语句中的隐式转换

  Boolean类型的转换规则

数据类型 转换为true的值 转换为false的值
String 任何非空字符串 空字符串
Number 任何非零数字值 NaN和0
Object 任何对象 Null
Undefined 无 undefined
  • Number类型
    分为浮点型和整型

  1.NaN（Not a number）,即非数值，是一个特殊的值，这个数值用来表示一个本来要返回的数值的操作数未返回数值的情况。比如，在其他语言中，任何数值除以0都会导致错误而终止程序的执行。但在ECMAScript中，会返回NaN，不会影响程序执行。

  2.任何与NaN的运算都等于NaN，而NaN与N任何值不相等

  3.把非数值转换为数值的三个方法：Number(),parseInt(),parseFloat().Number()可以用于任何的数据类型，而另外两个则专门用于字符串转换成数值

  • function类型

  2. call，apply方法（第一个参数指在h函数运行时指定的this值，用来确定作用域）

对于apply和call两者在作用上是相同的，但两者在参数上有区别的。对于第一个参数意义都一样，但对第二个参数：apply传入的是一个参数数组，也就是将多个参数组合成为一个数组传入，而call则作为call的参数传入（从第二个参数开始），如 func.call(func1,var1,var2,var3)对应的apply写法为：func.apply(func1, [var1,var2,var3])。同时，使用apply的好处是可以直接将当前函数的 arguments对象 作为apply的第二个参数传入。

变量，作用域和内存

  • 基本类型(Undifined,Null,Boolean,String,Number)，他们的值保存在栈内存中，有固定大小，按值访问
  • 向函数传递参数时,引用类型的值会把该值在内存中的地址复制给函数一个局部变量，因此这个局部变量的变化会反映在函数的外部，而值为基本类型的话则直接按值传递。

  • 初始化变量没有用var声明，该变量会自动添加到全局变量中

  • 引用类型(Object),必须在堆内存中分配空间，他们的值大小不固定，所以不能保存在栈内存中，但是他们内存地址固定，可以保持在栈内存中，这样当查询引用类型的变量时，可以先在栈内存中获取内存地址，在根据地址在堆内存中得到值，这种方式叫做引用访问。

  • if，for语句没有块作用域的功能，if,for中的变量是全局变量，没有作用域
  • 最外层的变量和方法属于window的属性和方法
  • 全局执行环境为Window对象
  • 优化内存最佳方案:一旦数据不再有用，将其设置为NULL，这叫做解除引用。

  • JS中内存的分配和回收都是自动完成的，内存在不使用的时候会被垃圾回收器自动回收

    1. 通过原型链实现的，即将父类的实例给子类的原型（不能传参）
      function father(){}
      functoin child(){}
      child.prototype=new father();
      对于实例和原型中的属性取值，使用的就近原则，实例中存在立即返回，不存在再查找原型
      1. 对象冒充继承
        使用call(apply)方法（可传参，但不能共享,只继承构造函数中的，不能继承原型中的）

匿名函数与闭包

  • 闭包
  1. 闭包就是指有权访问另一个函数作用域中的变量的函数
  2. 创建闭包的一种方式就是在一个函数内部创建另一个函数
  3. 通过闭包可以访问函数的局部变量，减少全局变量的使用（全局变量污染导致应用程序不可预测性，每个模块都可调用必将引来灾难，所以推荐使用私有的，封装的局部变量）
  4. 闭包内的this指向的是Window对象
  • 使用匿名函数创建块级作用域

可以避免向全局作用域添加过多的变量，造成命名空间污染。
这种做法可以减少闭包占用内存的问题，因为没有指向匿名函数的引用，只要函数执行完毕，就立马销毁其作用域链

  • 私有变量

在函数在定义的变量，都是私有变量，包括局部变量，函数参数，函数内定义的其他函数，函数外部正常访问不到

可以利用闭包来创建特权方法来访问这些私有变量
